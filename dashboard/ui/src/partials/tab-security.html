🛡️ 安全管理
按分层模型配置安全能力：先边界、再权限、再沙箱、最后韧性
📌 生效方式与操作建议
1) 当前页默认通过“保存并重启”生效，确保策略在所有 Agent/组件上一致。
2) 系统存在配置热加载能力（仅部分项可靠，例如 PathGuard 路径刷新），不建议把热加载当作唯一生效方式。
3) 推荐顺序：L0 边界 → L1 权限 → L2 沙箱 → L3 韧性。
Gateway 访问保护
文件路径沙箱
L0 边界层（入口认证）
目标：先阻止未授权请求进入系统。
L1 权限层（能力控制）
目标：明确“允许做什么/禁止做什么”，避免能力越权。
L2 沙箱层（作用域限制）
目标：将文件系统访问限制到业务必要目录，避免“*”全盘权限。
L3 韧性层（故障与滥用防护）
目标：提升长任务稳定性，防止卡死循环和无效重试。
推荐落地路径（给管理员）
• 第一步：先配置 L0（GATEWAY_API_KEY + WEBHOOK_HMAC_SECRET）。
• 第二步：收敛 L1（PERMISSION_POLICY + BLOCKED_TOOLS）。
• 第三步：收敛 L2（把 ALLOWED_PATHS 从 * 改成业务目录白名单）。
• 第四步：调优 L3（CHECKPOINT/重试阈值/步数上限），最后保存并重启验证。
